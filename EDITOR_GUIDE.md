# 📝 在线编辑器使用指南

## 概述

在线编辑器允许您在登录后直接在浏览器中编辑网站的所有文件（HTML、CSS、JavaScript、Markdown），并将修改提交到 GitHub 仓库。

---

## 🌟 主要功能

### 1. 文件浏览器
- 📁 以树形结构展示所有文件
- 📂 支持目录层级显示
- 🔍 快速定位和选择文件
- 🎨 不同类型文件图标区分

### 2. 代码编辑器
- ✏️ 在线编辑 HTML、CSS、JS、MD 文件
- 🔢 实时显示光标位置（行号、列号）
- 💾 本地自动保存功能
- ⌨️ 支持 Ctrl+S 快捷键保存

### 3. 实时预览
- 👁️ HTML 文件即时预览
- 🎨 CSS 样式效果预览
- 📱 预览面板可切换显示/隐藏

### 4. GitHub 集成
- ☁️ 直接提交修改到 GitHub
- 📝 自定义提交信息
- 🔄 自动获取最新版本
- ✅ 提交状态实时反馈

---

## 🚀 使用流程

### 第一步：登录系统

1. 访问 `login-secure.html`
2. 输入管理员用户名和密码
3. 点击"登录"按钮
4. 登录成功后可访问编辑器

### 第二步：配置 GitHub Token（首次使用）

**重要**：首次使用前必须配置 GitHub Token 才能提交修改。

1. 登录后访问 `github-config.html`
2. 创建 GitHub Personal Access Token：
   - 访问 https://github.com/settings/tokens
   - 点击 "Generate new token (classic)"
   - 填写备注（例如：Website Editor）
   - 设置过期时间（建议 30-90 天）
   - 勾选权限：**repo**（完整控制仓库）
   - 点击 "Generate token"
   - **复制生成的 token（只显示一次！）**

3. 填写配置信息：
   - **GitHub Token**：粘贴刚才复制的 token
   - **仓库所有者**：您的 GitHub 用户名（如：badhope）
   - **仓库名称**：github.io
   - **分支名称**：main（或 master）

4. 点击"保存配置"
5. 点击"测试连接"验证配置是否正确

### 第三步：编辑文件

1. 访问 `editor.html` 进入编辑器
2. 在左侧文件树中选择要编辑的文件
3. 在右侧编辑器中修改内容
4. 修改会自动保存到本地（浏览器存储）

### 第四步：预览效果

1. 点击"👁 预览"按钮
2. 右侧会显示预览面板
3. HTML 文件会直接渲染效果
4. CSS 文件会显示样式示例

### 第五步：提交到 GitHub

1. 点击"☁️ 提交到 GitHub"按钮
2. 输入提交信息（如：更新首页样式）
3. 点击确认
4. 等待提交完成提示

## 🔧 高级功能

### 本地保存
- 修改会自动保存到浏览器 LocalStorage
- 即使关闭浏览器也不会丢失
- 下次打开时自动恢复

### 快捷键
- **Ctrl + S**：快速保存（本地）
- **Ctrl + Z**：撤销
- **Ctrl + Y**：重做

### 状态指示
- **●** 橙色圆点：表示有未保存的修改
- 状态栏显示当前文件路径
- 实时显示光标位置

## 🔐 安全说明

### Token 安全
- ✅ Token 仅存储在您的浏览器本地
- ✅ 不会上传到任何第三方服务器
- ✅ 仅用于访问您的 GitHub 仓库
- ⚠️ 请定期更新 Token（建议每 30-90 天）

### Token 权限
建议的权限设置：
- ✅ **repo** - 完整控制私有仓库
- ❌ 不要授予 admin 权限
- ❌ 不要授予其他不相关的权限

### 访问控制
- 只有登录的管理员才能访问编辑器
- 游客无法访问编辑功能
- 会话超时后自动退出

## ⚠️ 注意事项

### 1. 提交前检查
- 确认修改内容正确
- 建议在本地预览后再提交
- 填写清晰的提交信息

### 2. 并发编辑
- 如果多人编辑，可能发生冲突
- 提交前会检查文件版本
- 如有冲突会提示重新加载

### 3. 文件大小
- 建议编辑小于 1MB 的文件
- 大文件可能导致浏览器卡顿
- 超大文件可能无法提交

### 4. 网络要求
- 需要稳定的网络连接
- 提交时需要访问 GitHub API
- 网络中断可能导致提交失败

## 🐛 故障排除

### 问题 1：无法加载文件列表
**原因**：
- GitHub Token 无效或过期
- 网络连接问题
- 仓库名称或用户名错误

**解决方法**：
1. 检查 Token 是否有效
2. 重新配置 GitHub Token
3. 检查网络连接

### 问题 2：提交失败
**可能原因**：
- Token 权限不足
- 文件已被他人修改
- 网络问题

**解决方法**：
1. 检查 Token 权限设置
2. 重新加载文件获取最新版本
3. 检查网络连接后重试

### 问题 3：预览不显示
**可能原因**：
- HTML 代码有语法错误
- 外部资源加载失败
- 浏览器安全限制

**解决方法**：
1. 检查 HTML 代码是否正确
2. 避免使用外部 CDN 资源
3. 在浏览器控制台查看错误

### 问题 4：编辑器无法访问
**原因**：未登录或会话过期

**解决方法**：
1. 访问 `login-secure.html` 重新登录
2. 登录后即可访问编辑器

## 🎯 快速开始示例

### 修改首页标题

1. 登录：访问 `login-secure.html`
2. 进入编辑器：访问 `editor.html`
3. 选择文件：点击左侧 `index.html`
4. 找到标题：搜索 `<title>` 标签
5. 修改内容：`<title>我的新标题</title>`
6. 预览效果：点击"👁 预览"查看效果
7. 提交修改：点击"☁️ 提交到 GitHub"
8. 填写信息：`更新首页标题`
9. 确认提交

### 添加新的 CSS 样式

1. 登录并进入编辑器
2. 选择 `css/style.css`
3. 在文件末尾添加新样式：
   ```css
   .my-new-style {
       color: blue;
       font-size: 16px;
   }
   ```
4. 预览效果
5. 提交：`添加新的样式类`
