React Hooks 深度解析：从入门到精通

👤熊泽城
📅2026-03-05
👁234 阅读
⏱12 分钟

React Hooks 是 React 16.8 引入的全新特性，它让你无需编写 class 组件就可以使用 state 和其他 React 特性。本文将深入讲解 React Hooks 的核心概念和最佳实践。

一、为什么需要 Hooks？

在 Hooks 出现之前，React 组件主要使用 class 来编写。这种方式存在以下问题：

  • 状态逻辑复用困难：class 组件之间共享状态逻辑需要使用高阶组件或 render props，导致代码嵌套复杂
  • 生命周期复杂：相关的逻辑分散在不同的生命周期方法中，难以维护
  • this 指向问题：class 中的 this 指向经常让开发者困惑

二、useState：状态管理

useState 是最基础的 Hook，用于在函数组件中添加状态：

import React, { useState } from 'react';

function Counter() {
  const [count, setCount] = useState(0);
  
  return (
    <div>
      <p>你点击了 {count} 次</p>
      <button onClick={() => setCount(count + 1)}>
        点击我
      </button>
    </div>
  );
}

三、useEffect：副作用处理

useEffect 用于处理函数组件中的副作用，比如数据获取、订阅或手动修改 DOM：

import React, { useState, useEffect } from 'react';

function Example() {
  const [data, setData] = useState(null);
  
  useEffect(() => {
    fetchData().then(result => {
      setData(result);
    });
    
    return () => {
      // 清理函数
    };
  }, []);
  
  return <div>{data}</div>
}

💡 提示：useEffect 的第二个参数是依赖数组，控制 effect 何时重新运行。空数组表示只在组件挂载和卸载时运行。

四、自定义 Hooks

你可以创建自己的 Hooks 来复用状态逻辑：

function useLocalStorage(key, initialValue) {
  const [storedValue, setStoredValue] = useState(() => {
    const item = window.localStorage.getItem(key);
    return item ? JSON.parse(item) : initialValue;
  });
  
  const setValue = (value) => {
    setStoredValue(value);
    window.localStorage.setItem(key, JSON.stringify(value));
  };
  
  return [storedValue, setValue];
}

五、最佳实践

1. 只在顶层调用 Hooks

不要在循环、条件或嵌套函数中调用 Hooks，确保它们在每次渲染时以相同的顺序被调用。

2. 使用 ESLint 插件

安装 eslint-plugin-react-hooks，它会自动检查 Hooks 的使用规则。

3. 合理拆分自定义 Hooks

将相关的逻辑封装到自定义 Hook 中，提高代码复用性和可测试性。
